1. Limit
반환되는 결과의 수를 조절할 수 있다.
order by와 연동하여 사용하는 경우가 많다. ex) 출간연도가 가장 빠른 5개의 행을 검색한다.

limit 1,2
1 => 시작하는 행
2 => 행의 개수(반환하는 개수)

만약 테이블의 행이 20개인데 limit을 50을 설정하면?
=> 오류가 발생하지 않고 최대한 많이 뽑아낸다. => 20개 나온다.

ex) select author_fname, title, stock_quantity from books order by author_fname, title limit 0,10;
=> order by가 된 데이터에 대해 limit를 뽑아낸다. 

Limit의 위치는 where 뒤나, order by 뒤에 나온다./ select 명령의 마지막에 위치된다.

만약 인자가 하나라면 숫자에 해당하는 행의 개수를 불러온다.
3 => 3개의 행 뽑아라 1,2,3

만약 인자가 두 개라면 첫 인자는 인덱스로 변해서 0=1이 되고, 두 번쨰 인자는 행의 개수가 된다.
(0,5) => 1부터 5개의 행을 뽑아라 => 1,2,3,4,5

2. like
데이터 검색에 유용한 키워드
ex) where로 'david'를 찾는다면 정확히 david만 나온다. 그러나 대부분의 검색어들은 모호한 방식으로 들어온다.
이때 유용한 키워드가 like이다.

like에 조건을 줄 떄 사용할 수 있는 키워드 = 와일드카드
%와_가 있다.

%
1. like 'a%' => a로 시작하는 문자열을 찾아라
2. like '%a' => a로 끝나는 문자열을 찾아라
3. like '%a%' => 문자열 안에 a가 들어있는 문자열을 찾아라.

_ => 문자열 하나를 의미한다.
1. like '_a' => ba,ca,ka/ 두글자를 찾는데 마지막 글자는 a여야 한다.
2. like 'a_' => ab,ac,ka/ 두글자를 찾는데 첫글자는 a여야 한다.
3. like '_a_' => bac, cac, kak/ 세글자를 찾는데 두 번째 글자는 a여야 한다.
4. like '_ _ _ _' => 문자가 4개를 문자열을 찾는다.

like는 항상 where절에 사용한다.

데이터가 '%10'인 경우 와일드카드 %와 겹친다.
이 때 %를 찾는 방법은 \역슬래시를 이용하는 것이다.
select title from books where title like '%\%%'
역슬래시는 항상 찾고자하는 문자 앞에 나와야 한다. \는 항상 앞에 있는 것을 문자열 취급하기 때문이다.
